Edo-Sign33 — це демонстраційна система електронного підпису, що складається з двох окремих .NET-застосунків:

1. EdoAuthServer (порт 7090)

Сервер автентифікації на базі Duende IdentityServer 6.
Забезпечує:

SSO (Single Sign-On)

видачу токенів OpenID Connect / OAuth2

авторизацію користувача

конфігурацію клієнтів і scopes

Використовує ASP.NET Identity та in-memory конфігурацію.

2. EdoSign.Lab-3 

Основний MVC веб-застосунок, який:

підключається до SSO через OpenID Connect

виконує автентифікацію користувача через EdoAuthServer

зберігає дані у SQLite

надає функцію електронного підпису документів

Підпис реалізовано через окрему бібліотеку.

3. NuGet-бібліотека EdoSign.Signing

Власна криптографічна бібліотека з реалізацією:

RSA-підпису

перевірки підпису

генерації ключів

Підключена до основного сайту через NuGet.

4. Деплой у Vagrant

Система розгортається у Ubuntu через Vagrant.
Встановлені:

.NET 9 (для MVC)

.NET 6 (для AuthServer)

Підсумок

Проєкт демонструє роботу SSO, OAuth2/OpenID Connect, авторизацію користувача та електронний підпис документів, використовуючи розділену архітектуру (AuthServer + MVC App) та власну криптобібліотеку.
